# EditLite 组件使用指南

本文档介绍 EditLite 内置可视化组件的使用方法。所有组件通过 Markdown 指令语法调用。

## 基本语法

```
:组件名{参数1="值1" 参数2="值2"}
```

**重要说明**：所有组件都有内置默认数据，无需手动输入数据即可运行。例如：
- `:graph{algorithm="dijkstra"}` - 使用默认图结构演示 Dijkstra 算法
- `:sort{algorithm="quick"}` - 使用默认数组演示快速排序
- `:search{algorithm="binary"}` - 使用默认有序数组演示二分查找

---

## 一、排序算法 `:sort{...}`

可视化展示排序算法执行过程。

### 支持的算法

| 算法 | 参数值 | 时间复杂度 |
|------|--------|-----------|
| 冒泡排序 | `bubble` | O(n²) |
| 快速排序 | `quick` | O(n log n) |
| 归并排序 | `merge` | O(n log n) |
| 插入排序 | `insertion` | O(n²) |
| 选择排序 | `selection` | O(n²) |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `algorithm` | string | `"bubble"` | 排序算法类型 |
| `array` | number[] | `[5,2,8,1,9,3,7,4,6]` | 待排序数组 |
| `speed` | number | `300` | 动画速度（毫秒） |
| `showSteps` | boolean | `true` | 是否显示步骤说明 |

### 使用示例

```
:sort{algorithm="bubble" array=[5,2,8,1,9,3,7,4,6]}

:sort{algorithm="quick" array=[10,7,8,9,1,5] speed=200}

:sort{algorithm="merge"}
```

## 二、查找算法 `:search{...}`

可视化展示查找算法执行过程。

### 支持的算法

| 算法 | 参数值 | 特点 |
|------|--------|------|
| 线性查找 | `linear` | 适用于无序数组 |
| 二分查找 | `binary` | 要求数组有序 |
| 插值查找 | `interpolation` | 要求数组有序，分布均匀时更高效 |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `algorithm` | string | `"binary"` | 查找算法类型 |
| `target` | number | `13` | 查找目标值 |
| `array` | number[] | `[1,3,5,7,9,11,13,...]` | 查找数组（需有序） |
| `speed` | number | `500` | 动画速度（毫秒） |

### 使用示例

```
:search{algorithm="binary" target=13}

:search{algorithm="linear" target=7 array=[5,3,8,1,9,7,2]}

:search{algorithm="interpolation" target=50 array=[10,20,30,40,50,60,70,80,90]}
```

### 颜色说明

- 🟡 **黄色**: 当前检查的元素
- 🟣 **紫色**: 当前搜索范围内的元素
- 🟢 **绿色**: 找到的目标元素

## 三、图算法 `:graph{...}`

可视化展示图算法执行过程。

**注意**：图算法组件内置了默认图结构（6个节点，9条边），无需手动指定 `nodes` 和 `edges` 即可运行。

### 支持的算法

| 算法 | 参数值 | 用途 |
|------|--------|------|
| 广度优先搜索 | `bfs` | 层级遍历、最短路径（无权） |
| 深度优先搜索 | `dfs` | 路径探索、连通性检测 |
| Dijkstra | `dijkstra` | 单源最短路径（带权） |
| Kruskal | `kruskal` | 最小生成树 |
| Prim | `prim` | 最小生成树 |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `algorithm` | string | `"bfs"` | 图算法类型 |
| `startNode` | string | `"A"` | 起始节点 ID |
| `nodes` | object[] | 预设6个节点 | 节点列表（可选） |
| `edges` | object[] | 预设9条边 | 边列表（可选） |
| `speed` | number | `800` | 动画速度（毫秒） |

### 快速使用（使用默认数据）

```
:graph{algorithm="bfs"}
:graph{algorithm="dfs"}
:graph{algorithm="dijkstra"}
:graph{algorithm="kruskal"}
:graph{algorithm="prim"}
```

### 指定起始节点

```
:graph{algorithm="bfs" startNode="B"}
:graph{algorithm="dijkstra" startNode="C"}
```

### 自定义图结构（高级用法）

```
:graph{
  algorithm="dijkstra"
  startNode="1"
  nodes=[{id="1"},{id="2"},{id="3"}]
  edges=[{from="1",to="2",weight=5},{from="2",to="3",weight=3}]
}
```

### 节点颜色说明

- ⚪ **默认**: 未访问节点
- 🟢 **绿色**: 已访问节点
- 🔵 **蓝色**: 当前处理节点

## 四、曲线图 `:chart{...}`

绘制数据曲线图，支持鼠标悬停交互。

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `title` | string | - | 图表标题 |
| `data` | number[] 或 object[] | - | 数据（必填） |
| `labels` | string[] | `["1","2",...]` | X轴标签 |
| `width` | number | `400` | 图表宽度 |
| `height` | number | `200` | 图表高度 |
| `showGrid` | boolean | `true` | 是否显示网格 |
| `showDots` | boolean | `true` | 是否显示数据点 |

### 单系列数据

```
:chart{
  title="销售趋势"
  data=[10, 25, 18, 32, 28, 45]
  labels=["一月", "二月", "三月", "四月", "五月", "六月"]
}
```

### 多系列数据

```
:chart{
  title="对比分析"
  data=[
    {values=[10, 20, 30, 40], label="产品A"},
    {values=[15, 25, 20, 35], label="产品B", color="#16a34a"}
  ]
  labels=["Q1", "Q2", "Q3", "Q4"]
}
```

### 交互功能

- 鼠标悬停显示垂直参考线
- 数据点放大效果
- 底部显示具体数值

## 五、树结构 `:tree{...}`

可视化展示树结构的插入、搜索操作。

### 支持的树类型

| 类型 | 参数值 | 特点 |
|------|--------|------|
| 二叉树 | `binary` | 基础树结构 |
| 二叉搜索树 | `bst` | 左小右大，支持高效查找 |
| AVL树 | `avl` | 自平衡，高度差不超过1 |
| 红黑树 | `redblack` | 自平衡，节点染色 |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `type` | string | `"bst"` | 树类型 |
| `values` | number[] | `[50,30,70,20,40,60,80]` | 初始节点值 |
| `insertValue` | number | 随机 | 要插入的值 |
| `searchValue` | number | 随机 | 要搜索的值 |
| `speed` | number | `800` | 动画速度（毫秒） |

### 使用示例

```
:tree{type="binary"}
:tree{type="bst"}
:tree{type="avl"}
:tree{type="redblack"}

:tree{type="bst" values=[10,5,15,3,7,12,20]}
:tree{type="avl" insertValue=25}
```

### 节点颜色说明

- 🔵 **蓝色**: 当前操作节点
- 🟡 **黄色**: 比较中
- 🟢 **绿色**: 找到目标
- 🟣 **紫色**: 旋转操作

## 六、堆 `:heap{...}`

可视化展示堆结构及其操作。

### 支持的堆类型

| 类型 | 参数值 | 特点 |
|------|--------|------|
| 最大堆 | `max` | 父节点 ≥ 子节点 |
| 最小堆 | `min` | 父节点 ≤ 子节点 |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `type` | string | `"max"` | 堆类型 |
| `values` | number[] | `[50,30,70,20,40,60,80,10,25]` | 初始元素 |
| `insertValue` | number | 随机 | 要插入的值 |
| `speed` | number | `600` | 动画速度（毫秒） |

### 使用示例

```
:heap{}
:heap{type="min"}
:heap{type="max" values=[100,50,75,25,60]}
```

### 操作说明

- **插入**: 添加元素到末尾，然后上浮调整
- **移除堆顶**: 删除根节点，用末尾元素替代，然后下沉调整

## 九、链表 `:linkedlist{...}`

可视化展示链表结构及操作。

### 支持的链表类型

| 类型 | 参数值 | 特点 |
|------|--------|------|
| 单链表 | `singly` | 单向指针，只能向后遍历 |
| 双链表 | `doubly` | 双向指针，可前后遍历 |
| 环形链表 | `circular` | 尾节点指向头节点 |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `type` | string | `"singly"` | 链表类型 |
| `values` | number[] | `[10,20,30,40,50]` | 初始节点值 |
| `insertValue` | number | 随机 | 要插入的值 |
| `searchValue` | number | 随机 | 要搜索的值 |
| `speed` | number | `500` | 动画速度（毫秒） |

### 使用示例

```
:linkedlist{}
:linkedlist{type="doubly"}
:linkedlist{type="circular" values=[1,2,3,4,5]}
```

### 操作说明

- **遍历**: 遍历整个链表
- **搜索**: 查找指定值的节点
- **头插**: 在链表头部插入节点
- **尾插**: 在链表尾部插入节点
- **删除**: 删除指定值的节点

## 十、哈希表 `:hashtable{...}`

可视化展示哈希表结构及冲突解决方法。

### 支持的冲突解决方法

| 方法 | 参数值 | 特点 |
|------|--------|------|
| 链地址法 | `chaining` | 冲突时使用链表存储 |
| 线性探测 | `linear` | 冲突时顺序查找下一个空位 |
| 二次探测 | `quadratic` | 冲突时使用二次函数计算偏移 |

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `method` | string | `"chaining"` | 冲突解决方法 |
| `size` | number | `8` | 哈希表容量 |
| `items` | object[] | 示例数据 | 初始键值对 |
| `insertKey` | string | 随机 | 要插入的键 |
| `insertValue` | number | 随机 | 要插入的值 |
| `searchKey` | string | 随机 | 要搜索的键 |
| `speed` | number | `600` | 动画速度（毫秒） |

### 使用示例

```
:hashtable{}
:hashtable{method="linear" size=10}
:hashtable{method="chaining" items=[{key="apple",value=5},{key="banana",value=8}]}
```

### 颜色说明

- 🟡 **黄色**: 正在探测/比较
- 🟢 **绿色**: 找到/插入成功
- 🔴 **红色**: 冲突

## 常见问题

### Q: 组件不显示？

检查语法格式：
1. 指令必须以 `:` 开头
2. 参数必须用花括号 `{}` 包裹
3. 字符串参数需要用双引号
4. 数组参数使用方括号

### Q: 如何调整动画速度？

使用 `speed` 参数，单位为毫秒，值越小越快：
```
:sort{speed=100}   # 快速
:sort{speed=500}   # 中速
:sort{speed=1000}  # 慢速
```

### Q: 图算法如何自定义节点位置？

当前版本使用预设布局，自定义位置功能将在后续版本支持。

## 开发者指南：插件接口规范

如果你想开发自定义可视化组件，请遵循以下接口规范：

### 统一接口

所有可视化组件必须实现 `VisualizerProps<T>` 接口：

```typescript
import type { VisualizerProps } from '../../core/types/directive';

interface MyConfig {
  // 组件特定的配置参数
  myParam?: string;
  speed?: number;
}

const MyVisualizer: React.FC<VisualizerProps<MyConfig>> = ({ args, onStateChange, onError }) => {
  const { myParam = 'default', speed = 500 } = args || {};

  // 组件实现...

  // 状态变化时调用
  onStateChange?.({ status: 'playing', currentStep: 1, totalSteps: 10 });

  return <div>...</div>;
};

export default MyVisualizer;
```

### 接口说明

| 属性 | 类型 | 说明 |
|------|------|------|
| `args` | `T` | 组件配置参数，必须有默认值 |
| `onStateChange` | `(state: VisualizationState) => void` | 可选，通知父组件状态变化 |
| `onError` | `(error: Error) => void` | 可选，错误处理回调 |

### 默认数据规范

**重要**：所有组件必须提供内置默认数据，确保用户无需输入任何参数即可正常运行：

```typescript
const MyVisualizer: React.FC<VisualizerProps<MyConfig>> = ({ args }) => {
  const {
    data = [1, 2, 3, 4, 5],  // 默认数据
    speed = 500,              // 默认速度
    title = '示例标题',        // 默认标题
  } = args || {};

  // ...
};
```

### 注册组件

在 `src/plugins/component-registry.ts` 中注册：

```typescript
registerComponent({
  name: 'mycomponent',
  loader: () => import('../visualizers/MyVisualizer'),
  description: '我的自定义组件',
  category: 'algorithm',
});
```
